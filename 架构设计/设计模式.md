## 一、创建型模式

### 1.1 单例模式 ⭐⭐⭐⭐⭐

**面试必问：手写线程安全单例。**

**双重检查锁（推荐）：**

```java
public class Singleton {
    // volatile 防止指令重排（new 分三步：分配内存→初始化→赋引用）
    private static volatile Singleton instance;

    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {                  // 第一次检查：无锁快速返回
            synchronized (Singleton.class) {
                if (instance == null) {          // 第二次检查：防止重复创建
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

**静态内部类（更优雅）：**

```java
public class Singleton {
    private Singleton() {}

    private static class Holder {
        // JVM 类加载机制保证线程安全且延迟加载
        static final Singleton INSTANCE = new Singleton();
    }

    public static Singleton getInstance() {
        return Holder.INSTANCE;
    }
}
```

**枚举（最简洁，防反射/序列化攻击）：**

```java
public enum Singleton {
    INSTANCE;
    public void doWork() { ... }
}
// 使用：Singleton.INSTANCE.doWork()
```

**框架应用：** Spring 默认 Bean 作用域（`singleton`）；`Runtime.getRuntime()`；线程池对象

### 1.2 工厂模式 ⭐⭐⭐⭐

**简单工厂（静态工厂）：**

```java
public class PaymentFactory {
    public static Payment create(String type) {
        return switch (type) {
            case "alipay"  -> new AlipayPayment();
            case "wechat"  -> new WechatPayment();
            default        -> throw new IllegalArgumentException("未知支付类型: " + type);
        };
    }
}
```

**抽象工厂（创建一系列相关对象）：**

```java
interface UIFactory {
    Button createButton();
    TextField createTextField();
}

class WindowsUIFactory implements UIFactory { ... }
class MacUIFactory implements UIFactory { ... }
// 切换操作系统只需换工厂，按钮和输入框风格自动匹配
```

**框架应用：** Spring `BeanFactory`；JDBC 的 `DriverManager.getConnection()`；Slf4j `LoggerFactory`

### 1.3 建造者模式 ⭐⭐⭐

**适用：** 构造参数多（超过4个），且部分可选时。

```java
// Lombok @Builder 原理等价于：
public class HttpRequest {
    private final String url;
    private final String method;
    private final Map<String, String> headers;
    private final String body;
    private final int timeout;

    private HttpRequest(Builder b) { ... }

    public static class Builder {
        private String url;
        private String method = "GET";
        private Map<String, String> headers = new HashMap<>();
        private String body;
        private int timeout = 5000;

        public Builder url(String url) { this.url = url; return this; }
        public Builder method(String method) { this.method = method; return this; }
        public Builder header(String k, String v) { headers.put(k, v); return this; }
        public Builder body(String body) { this.body = body; return this; }
        public Builder timeout(int ms) { this.timeout = ms; return this; }
        public HttpRequest build() { return new HttpRequest(this); }
    }
}

// 使用
HttpRequest req = new HttpRequest.Builder()
    .url("https://api.example.com")
    .method("POST")
    .header("Authorization", "Bearer token")
    .body("{}")
    .build();
```

**框架应用：** Lombok `@Builder`；`StringBuilder`；MyBatis `SqlSessionFactoryBuilder`

## 二、结构型模式

### 2.1 代理模式 ⭐⭐⭐⭐⭐

**静态代理：** 手动编写代理类，实现同一接口，组合被代理对象。

**JDK 动态代理（基于接口）：**

```java
// InvocationHandler：统一拦截所有方法
public class LogProxy implements InvocationHandler {
    private final Object target;
    public LogProxy(Object target) { this.target = target; }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        log.info("调用方法: {}", method.getName());
        long start = System.currentTimeMillis();
        Object result = method.invoke(target, args);
        log.info("耗时: {}ms", System.currentTimeMillis() - start);
        return result;
    }
}

// 生成代理对象
UserService proxy = (UserService) Proxy.newProxyInstance(
    target.getClass().getClassLoader(),
    target.getClass().getInterfaces(),
    new LogProxy(target)
);
```

**CGLIB 动态代理（基于子类，无需接口）：**

```java
// Spring AOP 当目标类无接口时自动切换 CGLIB
Enhancer enhancer = new Enhancer();
enhancer.setSuperclass(UserServiceImpl.class);
enhancer.setCallback((MethodInterceptor) (obj, method, args, proxy) -> {
    log.info("before: {}", method.getName());
    Object result = proxy.invokeSuper(obj, args);
    log.info("after: {}", method.getName());
    return result;
});
UserServiceImpl proxy = (UserServiceImpl) enhancer.create();
```

**框架应用：** Spring AOP（事务、缓存、日志）；MyBatis Mapper 接口；Dubbo RPC 调用

---

### 2.2 装饰器模式 ⭐⭐⭐

**与代理模式的区别：**
- **代理**：控制访问，增强行为（不由客户端选择）
- **装饰器**：动态叠加功能，客户端可自由组合

```java
// Java IO 典型装饰器链
InputStream is =
    new BufferedInputStream(          // 装饰：增加缓冲
        new GZIPInputStream(          // 装饰：增加解压
            new FileInputStream("file.gz")  // 原始组件
        )
    );
```

**框架应用：** Java IO(`BufferedInputStream`/`DataInputStream`)；Spring `HttpServletRequestWrapper`

### 2.3 适配器模式 ⭐⭐⭐

**目的：** 将不兼容的接口转换为目标接口。

```java
// 将第三方短信服务适配到统一 SmsService 接口
public class SmsAdapter implements SmsService {
    private final ThirdPartySmsClient client;  // 接口不兼容的第三方

    @Override
    public void send(String phone, String message) {
        client.sendMsg(new PhoneNumber(phone), message);  // 转换调用方式
    }
}
```

**框架应用：** Spring MVC `HandlerAdapter`（适配各种 Controller 类型）；`Arrays.asList()`（数组→List）

## 三、行为型模式

### 3.1 策略模式 ⭐⭐⭐⭐⭐

**目的：** 消除大量 if-else/switch，将算法封装为独立类，运行时切换。

```java
// 定义策略接口
public interface DiscountStrategy {
    BigDecimal calculate(BigDecimal price);
}

// 各策略实现
@Component("vipDiscount")
public class VipDiscountStrategy implements DiscountStrategy {
    public BigDecimal calculate(BigDecimal price) { return price.multiply(new BigDecimal("0.8")); }
}

@Component("couponDiscount")
public class CouponDiscountStrategy implements DiscountStrategy {
    public BigDecimal calculate(BigDecimal price) { return price.subtract(new BigDecimal("20")); }
}

// 策略注册（用 Spring 容器替代 if-else）
@Service
public class OrderService {
    @Autowired
    private Map<String, DiscountStrategy> strategies;  // Spring自动注入所有策略

    public BigDecimal calcPrice(String discountType, BigDecimal price) {
        DiscountStrategy strategy = strategies.get(discountType + "Discount");
        if (strategy == null) throw new BusinessException("未知优惠类型");
        return strategy.calculate(price);
    }
}
```

**框架应用：** Spring `Resource`（各种资源加载策略）；比较器 `Comparator`；线程池拒绝策略

---

### 3.2 观察者模式 ⭐⭐⭐⭐

**目的：** 一对多依赖，被观察者状态变更时自动通知所有观察者（事件/消息驱动的基础）。

```java
// Spring 事件（Spring ApplicationEvent 就是观察者模式实现）
// 定义事件
public class OrderCreatedEvent extends ApplicationEvent {
    public OrderCreatedEvent(Order source) { super(source); }
}

// 发布事件
@Service
public class OrderService {
    @Autowired
    private ApplicationEventPublisher publisher;

    public void createOrder(Order order) {
        save(order);
        publisher.publishEvent(new OrderCreatedEvent(order));  // 解耦后续处理
    }
}

// 监听事件（多个监听器互相独立）
@EventListener
public void sendNotification(OrderCreatedEvent event) { ... }

@EventListener
@Async  // 异步执行
public void deductInventory(OrderCreatedEvent event) { ... }
```

**框架应用：** Spring `ApplicationEvent`；JDK `Observable`；MQ（Kafka/RocketMQ 的订阅发布）

### 3.3 模板方法模式 ⭐⭐⭐⭐

**目的：** 父类定义算法骨架，子类实现具体步骤，复用整体流程。

```java
// 父类：定义不变骨架
public abstract class DataImporter {
    // 模板方法（final 防止被覆盖）
    public final void importData(String source) {
        List<String> rawData = readData(source);   // 1. 读取（可变）
        List<Object> parsed = parseData(rawData);  // 2. 解析（可变）
        validate(parsed);                          // 3. 校验（固定）
        saveData(parsed);                          // 4. 保存（固定）
    }

    protected abstract List<String> readData(String source);
    protected abstract List<Object> parseData(List<String> rawData);

    private void validate(List<Object> data) { ... }    // 子类不可变
    private void saveData(List<Object> data) { ... }    // 子类不可变
}

// 子类：只实现差异化步骤
public class CsvImporter extends DataImporter {
    protected List<String> readData(String path) { /* 读CSV */ }
    protected List<Object> parseData(List<String> raw) { /* 解析CSV */ }
}
```

**框架应用：** Spring `JdbcTemplate`、`RedisTemplate`；AQS（`tryAcquire` 由子类实现）；Servlet `doGet/doPost`

---

### 3.4 责任链模式 ⭐⭐⭐⭐

**目的：** 请求沿处理器链传递，每个处理器决定处理还是传递给下一个。

```java
// Netty ChannelPipeline、Spring Security FilterChain 都是责任链
// 简单实现
public abstract class Handler {
    protected Handler next;
    public Handler setNext(Handler next) { this.next = next; return next; }

    public abstract void handle(Request req);
}

public class AuthHandler extends Handler {
    public void handle(Request req) {
        if (!req.hasToken()) { throw new AuthException(); }
        if (next != null) next.handle(req);  // 传递给下一个
    }
}

// Spring MVC 拦截器链
// Netty ChannelPipeline
// Spring Security FilterChain
```

**框架应用：** Netty `ChannelPipeline`；Spring MVC `HandlerInterceptor`；Spring Security 过滤器链；OkHttp `Interceptor`

## 四、常见面试追问

**Q: 代理模式和装饰器模式有什么区别？**

> **代理模式**：控制对目标对象的访问，代理类通常自己创建被代理对象（或注入），关注点是"是否允许/如何访问"（如权限控制、延迟加载、日志记录）。**装饰器模式**：为对象动态添加功能，装饰对象由外部传入（开放给客户端组合），关注点是"功能增强"，多个装饰器可以自由叠加（如 Java IO 流）。两者结构相似，区别在于**意图和组合方式**。

**Q: 工厂方法模式 vs 抽象工厂模式？**

> **工厂方法**：每个子类工厂只负责创建**一种**产品，扩展产品类型只需新增工厂子类，符合开闭原则。**抽象工厂**：每个子类工厂负责创建**一整套相互关联的产品族**（如 Windows 风格的按钮+输入框+对话框），切换产品族只需换工厂，但增加新的产品类型需要修改所有工厂（扩展性差）。简单说：工厂方法关注**单个产品的创建**，抽象工厂关注**一组相关产品的统一创建**。

**Q: 什么场景下用策略模式替代 if-else？**

> 当 if-else 满足以下任一条件时考虑策略模式：① 分支超过4-5个且持续增加；② 每个分支逻辑较复杂且独立；③ 分支类型需要在运行时动态决定；④ 不同分支逻辑可能在多处重复出现。策略模式配合 Spring 容器（Map 注入所有策略）可以做到新增策略只加类、不改已有代码，完美符合开闭原则。
